# System Design Brief

本文件只能根據已確認需求產生。所有設計決策追溯至 `specs/04-design-ready/requirement-spec.md`。

## Scope

員工請假管理系統：提供員工線上申請請假、主管審核、HR 查詢報表的 Web 應用程式。

## Architecture Overview

| 層次 | 說明 | 關聯需求 |
| --- | --- | --- |
| Frontend | Web 介面，依角色呈現不同頁面與操作 | FR-001 ~ FR-008、PR-001 ~ PR-007 |
| Backend API | RESTful API，處理業務邏輯與資料存取 | 全部 FR、BR、WR |
| Database | 儲存員工、請假申請、審核紀錄、假期餘額、代理設定 | DR-001 ~ DR-011 |
| Notification Service | 發送系統內通知與 Email | FR-004、BR-006、DEC-002 |
| Scheduler | 定期掃描超過 3 天未審核的申請並觸發催審 | FR-004、WR-004 |
| Report Generator | 產出 Excel 格式假期報表 | FR-006、DR-011 |

## Key Requirements Summary

| Requirement ID | 設計影響 |
| --- | --- |
| FR-001、BR-008 | 申請表單支援 0.5 天最小單位，需處理半天日期計算 |
| FR-002、BR-007 | 病假申請需條件性附件上傳（> 3 天才強制），附件儲存機制待確認（DR-005 未定義格式限制） |
| FR-003、BR-005 | 審核表單需強制填寫拒絕原因，並記錄完整審核紀錄 |
| FR-004、BR-006 | 需 Scheduler 定期執行，發現超時申請後同時觸發系統通知與 Email |
| FR-007、WR-005 | 取消邏輯需驗證申請狀態，只允許「待審核」狀態取消 |
| FR-008、BR-010、BR-013、BR-015 | 代理設定需驗證代理人不為自己、日期合法性，且代理到期後自動回歸原主管 |
| BR-011、BR-012 | 申請送出前需驗證餘額足夠且日期不重疊（兩個獨立驗證） |
| BR-009 | 審核時需依員工主要部門決定審核主管，不依登入主管自行判斷 |
| PR-002 | 員工查詢需過濾，僅回傳自身申請紀錄 |
| DR-011 | 報表匯出需整合 8 個固定欄位，格式為 Excel |

## Constraints

- 附件上傳的檔案格式與大小限制尚未在需求中確認（DR-005），設計時需保留彈性或回寫 open question
- 非功能需求（效能、安全、可用性）尚未確認，本版設計不作假設

## Out of Scope

- 薪資計算與薪資單產出
- 排班與工時管理
- 系統管理員後台
- 非功能需求（效能指標、SLA 等）
