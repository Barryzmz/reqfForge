## UC-001：員工申請請假

- **主要角色**：員工（Employee）
- **前置條件**：員工已登入系統
- **觸發條件**：員工選擇申請請假
- **基本流程**：
  1. 員工填寫開始日期、結束日期（最小單位 0.5 天）、假別、原因
  2. 若假別為病假且請假天數超過 3 天，員工需上傳診斷書
  3. 員工送出申請
  4. 系統建立請假申請，狀態設為「待審核」
- **替代流程**：
  - A1. 員工未填寫必填欄位 → 系統提示缺漏欄位，拒絕送出
  - A2. 員工申請病假超過 3 天但未上傳診斷書 → 系統提示需上傳診斷書，拒絕送出
- **後置條件**：系統存在一筆狀態為「待審核」的請假申請，等待主管審核
- **關聯需求**：FR-001、FR-002、BR-007、BR-008、DR-001 ~ DR-005

## UC-002：主管審核請假申請

- **主要角色**：主管（Manager）或代理主管
- **前置條件**：主管已登入系統；存在一筆屬於管轄員工且狀態為「待審核」的請假申請
- **觸發條件**：主管進入待審核申請列表並選擇一筆申請
- **基本流程（同意）**：
  1. 主管查看申請內容
  2. 主管選擇同意
  3. 系統將申請狀態更新為「已核准」，並記錄審核時間與審核人
- **替代流程（拒絕）**：
  - A1. 主管選擇拒絕 → 主管填寫拒絕原因 → 系統將申請狀態更新為「已拒絕」，並記錄拒絕原因、審核時間與審核人
  - A2. 主管選擇拒絕但未填寫原因 → 系統提示需填寫拒絕原因，拒絕送出
- **後置條件**：請假申請狀態更新為「已核准」或「已拒絕」，審核紀錄已儲存
- **關聯需求**：FR-003、BR-005、BR-009、BR-010、PR-004、PR-005、WR-002、WR-003

## UC-003：系統催審提醒

- **主要角色**：系統（System）
- **前置條件**：存在一筆狀態為「待審核」的請假申請
- **觸發條件**：請假申請送出後超過 3 天，申請狀態仍為「待審核」
- **基本流程**：
  1. 系統偵測到超過 3 天未審核的待審核申請
  2. 系統對負責審核的主管發送系統內通知
  3. 系統同時發送 Email 通知給該主管
  4. 申請狀態維持「待審核」不變
- **後置條件**：主管已收到催審通知；申請狀態不變
- **關聯需求**：FR-004、BR-006、WR-004

## UC-004：HR 查看全員假期紀錄

- **主要角色**：HR
- **前置條件**：HR 已登入系統
- **觸發條件**：HR 進入假期紀錄查詢頁面
- **基本流程**：
  1. HR 查看假期紀錄
  2. 系統顯示所有員工的請假申請紀錄
- **後置條件**：HR 可查看全員請假資料
- **關聯需求**：FR-005、PR-006

## UC-005：HR 匯出假期報表

- **主要角色**：HR
- **前置條件**：HR 已登入系統
- **觸發條件**：HR 選擇匯出報表
- **基本流程**：
  1. HR 選擇匯出報表
  2. 系統產出 Excel 格式報表，包含：員工姓名、部門、假別、開始日期、結束日期、天數、狀態、申請日期
  3. HR 下載報表檔案
- **後置條件**：HR 已取得 Excel 報表檔案
- **關聯需求**：FR-006、PR-007、DR-011

## UC-006：員工取消請假申請

- **主要角色**：員工（Employee）
- **前置條件**：員工已登入系統；目標申請狀態為「待審核」
- **觸發條件**：員工選擇取消一筆「待審核」狀態的申請
- **基本流程**：
  1. 員工查看自己的請假申請
  2. 員工選擇取消一筆「待審核」申請
  3. 系統將申請狀態更新為「已取消」
  4. 該申請從主管待審核清單移除
- **替代流程**：
  - A1. 員工嘗試取消「已核准」或「已拒絕」的申請 → 系統拒絕操作並提示無法取消已審核的申請
- **後置條件**：申請狀態為「已取消」，不再出現於主管待審核清單
- **關聯需求**：FR-007、PR-003、WR-005

## UC-007：主管設定代理主管

- **主要角色**：主管（Manager）
- **前置條件**：主管已登入系統
- **觸發條件**：主管設定代理主管資訊
- **基本流程**：
  1. 主管指定代理人（系統使用者）
  2. 主管設定代理起始日與結束日
  3. 系統儲存代理設定
  4. 代理期間內，代理人可審核被代理主管管轄員工的請假申請
- **替代流程**：
  - A1. 代理期間結束後，代理人嘗試審核相關申請 → 系統拒絕操作並提示代理期間已結束
- **後置條件**：代理設定已儲存；代理期間內代理人擁有對應審核權限
- **關聯需求**：FR-008、BR-010、PR-005、DR-010
