# Edge Cases

| Edge Case ID | 情境 | 預期處理 | 關聯需求 | 狀態 |
| --- | --- | --- | --- | --- |
| EC-001 | 員工申請天數超過剩餘假期餘額 | 系統應拒絕送出並提示餘額不足，顯示該假別剩餘天數 | DR-008、BR-001 ~ BR-004 | Confirmed |
| EC-002 | 員工申請病假恰好 3 天（邊界條件） | 3 天以內不強制診斷書，恰好 3 天視為「3 天以內」，不需附件 | FR-002、BR-007 | Confirmed |
| EC-003 | 員工申請病假 3.5 天（超過 3 天邊界，含半天） | 超過 3 天，需強制上傳診斷書 | FR-002、BR-007、BR-008 | Confirmed |
| EC-004 | 主管本身請假期間有員工送出請假申請，且主管未設定代理主管 | 申請維持「待審核」狀態；系統於超過 3 天後依 BR-006 觸發催審提醒，等待主管返回後審核 | FR-004、BR-006、WR-004 | Confirmed |
| EC-005 | 代理期間結束時仍有「待審核」申請未被代理主管處理 | 申請回歸被代理主管的待審核清單，由被代理主管繼續審核 | BR-010、PR-005 | Confirmed |
| EC-006 | 跨部門員工的主要部門主管本身也在代理期間，由誰審核 | 由代理主管審核（代理主管擁有與原主管相同的審核範圍） | BR-009、BR-010、PR-005 | Confirmed |
| EC-007 | 員工同一時段重複申請同一假別（日期重疊） | 系統應拒絕送出並提示申請日期與現有申請重疊 | FR-001、DR-001、DR-002 | Confirmed |
| EC-008 | 員工申請的結束日期早於開始日期 | 系統應拒絕送出並提示日期錯誤 | DR-001、DR-002 | Confirmed |
| EC-009 | 員工假期餘額剛好用完（剩餘 0 天），仍送出申請 | 系統應拒絕送出並提示餘額不足（剩餘 0 天）（同 EC-001 處理方式） | DR-008 | Confirmed |
| EC-010 | 申請喪假時選擇的親等不在系統支援的對照表範圍內 | 系統應拒絕送出並提示該親等不在支援範圍，建議聯繫 HR 處理 | BR-003 | Confirmed |
| EC-011 | 主管設定的代理結束日早於代理起始日 | 系統應拒絕儲存代理設定並提示日期錯誤 | DR-010 | Confirmed |
| EC-012 | 主管將自己設定為代理人（自我代理） | 系統應拒絕並提示代理人不可為自己 | FR-008、DR-010 | Confirmed |
